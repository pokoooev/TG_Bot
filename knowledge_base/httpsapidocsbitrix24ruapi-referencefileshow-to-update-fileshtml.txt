## ОПИСАНИЕ
В Битрикс24 есть два типа файловых полей. Файл. Поле не связано с Диском, в него файлы загружаются напрямую через строку формата Base64. После загрузки в поле хранится ID файла. Файл (диск). Поле связано с Диском, в поле хранится ID объекта диска. Поле не обрабатывает формат Base64. Для обновления файлов диска используйте методы disk.file.*. Если поле не множественное, загрузите новый файл в поле методом *.update. Используйте формат передачи данных Base64. При загрузке нового файла старый файл удалится автоматически. Чтобы очистить поле, передайте пустое значение. Если поле множественное, в поле хранится массив из ID файлов. При обновлении множественных полей типа «файл» учитывайте особенности методов. Для обновления полей в объектах CRM используйте универсальный метод crm.item.update. Не рекомендуется использовать методы crm.deal.update, crm.lead.update, crm.contact.update, crm.company.update для обновления файловых полей. Перед обновлением поля получите ID текущих файлов, чтобы сохранить их. Можно использовать метод crm.item.get, он вернет все поля элемента, или метод crm.item.list с выбором только нужного поля типа «файл» в select. В ответе получим информацию по файлам: ID и ссылки на скачивание. В зависимости от переданных параметров метод crm.item.update выполняет операции: загрузки новых файлов — передавайте контент в формате Bаse64, удаления старых файлов — не передавайте ID этих файлов в массиве, сохранения файлов — передавайте ID в массиве файлов. Файлы будут сохранены, если их ID перечислены в запросе. Файлы будут удалены, если их ID отсутствует в запросе. Чтобы удалить все файлы, передавайте пустой массив в поле. Для обновления файлов в комментариях элементов CRM используйте метод crm.timeline.comment.update. Старые файлы всегда удаляются при обновлении значения поля. Новые файлы загружайте в поле в формате Base64. Чтобы удалить все файлы, передавайте пустой массив в поле FILES. Чтобы загрузить новые файлы в поле элемента списка, передавайте файлы методом lists.element.update в формате Base64. Старые файлы останутся в поле без изменений. Для удаления файлов понадобится ID значения свойства. Чтобы получить ID для удаления файла, выполните метод lists.element.get, он вернет все поля элемента. Поле «файл» в примере — PROPERTY_1075. В поле получим информацию: первое значение "3693" — это ID значения свойства, второе значение "31219"— это ID файла. Передайте в метод lists.element.update поле с постфиксом _DEL, например PROPERTY_1075_DEL. В поле укажите список из ID значений свойств, которые будут удалены: ключ — ID значения свойства, значение — Y. Чтобы загрузить новые файлы к посту в ленте, передавайте файлы методом log.blogpost.update в формате Base64. Старые файлы останутся в посте без изменений. Для удаления файлов понадобится их ID. Чтобы получить ID для удаления файла, выполните метод log.blogpost.get, он вернет все поля поста, включая FILES. В ответе получим массив объектов: первое значение 0 — это порядковый ID файла в посте, второе значение 437— это ID файла. Передайте в метод log.blogpost.update поле FILES. В поле укажите массив ID файлов, которые будут удалены: ключ — ID файла, значение — del. Чтобы удалить все файлы из поста, передайте в метод log.blogpost.update поле UF_BLOG_POST_FILE. В значении поля укажите ["empty"]. Чтобы загрузить новые файлы в карточку товара, передавайте файлы методом catalog.product.update в формате Base64. Старые файлы останутся в поле без изменений. Для удаления файлов понадобится ID значения поля. Чтобы получить ID для удаления файла, выполните метод catalog.product.get. Метод вернет все поля товара. Поле «файл» в примере — property1077. Поле содержит массив объектов: value — это информация по файлу: ID и ссылки на скачивание, valueId — это ID значения поля. Чтобы удалить файл, передайте в метод catalog.product.update поле со значениями: value — укажите remove как ключ, Y как значение, valueId — укажитеID значения поля, файл которого будет удален.
