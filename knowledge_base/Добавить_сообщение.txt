# Добавить сообщение
URL: https://apidocs.bitrix24.ru/api-reference/log/log-blogpost-add.html

## ОПИСАНИЕ
Scope: log Кто может выполнять метод: любой пользователь Добавляет в Ленту новостей сообщение от имени текущего пользователя. Параметр Описание USER_ID ID автора записи (опционально, по умолчанию - текущий пользователь, другое значение доступно только администратору в коробочной версии). POST_MESSAGE Текст сообщения. POST_TITLE Заголовок сообщения. DEST Список адресатов, которые получат право на просмотр сообщения. Возможные значения элементов массива: Значение по умолчанию - ['UA'] SPERM Список адресатов, которые получат право на просмотр сообщения (устаревшее). Аналогично DEST FILES Файлы, массив значений, описанный по правилам. IMPORTANT По умолчанию N. Сообщение ленты публикуется как "важное". IMPORTANT_DATE_END Указывается значение даты/времени, до которого сообщение будет считаться важным. Обязательные параметры отмечены * Использование метода REST API log.blogpost.add (блог разработчика)

## ТАБЛИЦЫ

Таблица 0:
USER_ID | ID автора записи (опционально, по умолчанию - текущий пользователь, другое значение доступно только администратору в коробочной версии).
POST_MESSAGE | Текст сообщения.
POST_TITLE | Заголовок сообщения.
DEST | Список адресатов, которые получат право на просмотр сообщения. Возможные значения элементов массива:
SG<X> - рабочая группа/проект с идентификатором X;
U<X> - пользователь с идентификатором X (включая, extranet-пользователей) или UA - все авторизованные пользователи;
DR<X> - отдел с идентификатором X, включая подразделы.
Значение по умолчанию - ['UA']
SPERM | Список адресатов, которые получат право на просмотр сообщения (устаревшее). Аналогично DEST
FILES | Файлы, массив значений, описанный по правилам.
IMPORTANT | По умолчанию N. Сообщение ленты публикуется как "важное".
IMPORTANT_DATE_END | Указывается значение даты/времени, до которого сообщение будет считаться важным.
