## ОПИСАНИЕ
Быстрый переход: все методы и события События в Битрикс24 — это уведомления об изменениях данных, например, о создании сделки или удалении товара. Когда приложение или вебхук подписывается на событие, система начинает генерировать для него эти уведомления. Чтобы получать события, настройте обработчик. Обработчик события — это внешний URL-адрес, на который Битрикс24 отправляет POST-запрос с данными об изменении. Обработчик позволяет: URL обработчика должен быть доступен из внешней сети. Не используйте адреса на localhost или в локальной сети. Проверяйте доступность вашего URL с помощью публичных сервисов. События обрабатываются не напрямую. Сначала Битрикс24 ставит событие в очередь на специальном сервере. Оттуда на ваш обработчик отправляется POST-запрос. Из-за этого запрос может прийти с небольшой задержкой. Сервер проверяет скорость ответа обработчика. Если обработчик отвечает медленно, сервер снижает частоту вызовов. Интервалы между запросами увеличиваются. Актуальные адреса сервера очередей. Система отправляет запрос с content-type application/x-www-form-urlencoded. Основные ключи: Пример запроса: Токены OAuth 2.0 в запросе привязаны к пользователю, который совершил действие. Если событие вызывается автоматически, например, роботом или бизнес-процессом, идентификатор пользователя будет равен 0. В этом случае токены в обработчик не передаются. Чтобы приложение всегда могло делать обратные вызовы к Битрикс24, сохраните токены пользователя, который установил приложение. Используйте эти токены для любых последующих запросов от имени приложения. Пример запроса: Обратите внимание, что в запросе от исходящего вебхука отсутствуют пользовательские токены OAuth 2.0. У событий есть два основных ограничения: Если важно обработать все события без потерь, используйте офлайн-события. Они позволяют получить события из очереди вручную. Метод Описание event.bind Регистрирует новый обработчик события event.get Получает список зарегистрированных обработчиков событий event.offline.clear Очищает записи в очереди офлайн-событий event.offline.error Регистрирует ошибки обработки очереди офлайн-событий event.offline.get Получает список офлайн-событий с «очисткой» event.offline.list Получает список офлайн-событий event.unbind Отменяет зарегистрированный обработчик события events Получает список доступных событий
