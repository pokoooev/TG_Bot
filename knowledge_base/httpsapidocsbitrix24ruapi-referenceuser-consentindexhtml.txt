## ОПИСАНИЕ
В Битрикс24 можно создавать пользовательские соглашения. Они нужны, если вы обрабатываете персональные данные, отправляете рассылки или выполняете другие действия, которые требуют согласия пользователя. С помощью REST API можно: Создавать и изменять соглашения нужно через интерфейс Битрикс24. Быстрый переход: все методы В Битрикс24 есть два вида соглашений: стандартные и пользовательские. Стандартные соглашения создаются из предустановленных шаблонов. Они поддерживают локализацию и замену переменных через параметр replace в методе userconsent.agreement.text. Пользовательские соглашения создаются вручную. Они содержат произвольный HTML-текст и не поддерживают подстановку переменных. Для сбора согласий на обработку персональных данных используйте стандартное соглашение. Добавьте реквизиты компании и электронную почту для обратной связи. Для остальных сценариев создавайте пользовательские соглашения. Пользователи. Согласия можно связать с пользователями через параметр USER_ID. Чтобы получить идентификатор пользователя, используйте методы user.get и user.search. Пользовательская документация Scope: userconsent Кто может выполнять метод: любой пользователь Метод Описание userconsent.agreement.list Получает список соглашений userconsent.agreement.text Получает текст соглашения userconsent.consent.add Сохраняет полученное согласие пользователя
