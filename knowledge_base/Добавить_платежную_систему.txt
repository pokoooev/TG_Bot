# Добавить платежную систему
URL: https://apidocs.bitrix24.ru/api-reference/pay-system/sale-pay-system-add.html

## ОПИСАНИЕ
Scope: pay_system Кто может выполнять метод: администратор CRM (право «Разрешить изменять настройки») Метод добавляет платежную систему. Обязательные параметры отмечены * Название
тип Описание NAME*
string Название платежной системы DESCRIPTION
string Описание платежной системы PERSON_TYPE_ID*
sale_person_type.id Идентификатор типа плательщика BX_REST_HANDLER*
sale_paysystem_handler.CODE Код REST-обработчика, указанный при добавлении обработчика методом sale.paysystem.handler.add ACTIVE
string Индикатор активности платежной системы. Возможные значения: Если не передано, то по умолчанию N SETTINGS
object Список значений настроек обработчика в формате {"field_1": "value_1", ... "field_N": "value_N"}, где field — название настройки, а value — объект, содержащий ключи TYPE и VALUE (описание смотрите ниже). Структура настроек задается при добавлении обработчика платежной системы в методе sale.paysystem.handler.add в ключе CODES параметра SETTINGS ENTITY_REGISTRY_TYPE*
string Привязка платежной системы: LOGOTYPE
string Логотип платежной системы (картинка в формате Base64) NEW_WINDOW
string Флаг, отвечающий за настройку «Открывать в новом окне». Возможные значения: Если не передано, то по умолчанию N XML_ID
string Внешний идентификатор платежной системы. Может быть использован как дополнительный параметр для фильтрации в sale.paysystem.list Название Описание ORDER Параметры PROPERTY Свойства счета PAYMENT Оплата USER Пользователь VALUE Произвольное значение типа строка Y\N Флажок Название Описание ORDER PAYMENT USER Как использовать примеры в документации HTTP-статус: 200 Название
тип Описание result
integer Идентификатор добавленной платежной системы time
time Информация о времени выполнения запроса HTTP-статус: 400, 403 Название
тип Описание error
string Строковый код ошибки. Может состоять из цифр, латинских букв и знака подчеркивания error_description
error_description Текстовое описание ошибки. Описание не предназначено для показа конечному пользователю в необработанном виде Код Описание Статус ACCESS_DENIED Недостаточно прав для добавления платежной системы 403 ERROR_CHECK_FAILURE Не указано значение обязательного поля либо значение одного из полей указано неверно 400 ERROR_PAY_SYSTEM_ADD Прочие ошибки. Подробную информацию об ошибке смотрите в error_description 400 ERROR_HANDLER_NOT_FOUND Обработчик, указанный в параметре BX_REST_HANDLER, не найден 400 ERROR_PERSON_TYPE_NOT_FOUND Тип плательщика, указанный в параметре PERSON_TYPE_ID, не найден 400 HTTP-статус: 20х, 40x, 50x Описанные ниже ошибки могут возникнуть при вызове любого метода Статус Код

## ТАБЛИЦЫ

Таблица 0:
NAME*
string | Название платежной системы

string | Описание платежной системы

sale_person_type.id | Идентификатор типа плательщика
BX_REST_HANDLER*
sale_paysystem_handler.CODE | Код REST-обработчика, указанный при добавлении обработчика методом sale.paysystem.handler.add

string | Индикатор активности платежной системы. Возможные значения:

object | Список значений настроек обработчика в формате {"field_1": "value_1", ... "field_N": "value_N"}, где field — название настройки, а value — объект, содержащий ключи TYPE и VALUE (описание смотрите ниже).

string | Привязка платежной системы:
ORDER — значение для заказов магазина, сделок, смарт-процессов

string | Логотип платежной системы (картинка в формате Base64)

Y — да

string | Внешний идентификатор платежной системы. Может быть использован как дополнительный параметр для фильтрации в sale.paysystem.list
