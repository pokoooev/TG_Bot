## ОПИСАНИЕ
Scope: department Кто может выполнять метод: любой пользователь Метод department.get предназначен для получения списка подразделений. Обязательные параметры отмечены * Название
тип Описание sort
string Название поля, по которому нужна сортировка. Сортировка работает по полям: order
string Направление сортировки: ID
int Фильтр по идентификатору подразделения NAME
string Фильтр по имени подразделения. Указывается полное название подразделения SORT
int Фильтр по полю сортировки PARENT
int Фильтр по идентификатору родительского подразделения UF_HEAD
int Фильтр по идентификатору руководителя подразделения START
integer Параметр используется для управления постраничной навигацией. Размер страницы результатов всегда статичный: 50 записей. Чтобы выбрать вторую страницу результатов, необходимо передавать значение 50. Чтобы выбрать третью страницу результатов — значение 100 и так далее. Формула расчета значения параметра start: start = (N-1) * 50, где N — номер нужной страницы. Подробности в статье Особенности списочных методов Как использовать примеры в документации HTTP-статус: 200 Название
тип Описание result
department[] Корневой элемент ответа, который содержит отфильтрованный список подразделений total
integer Общее количество найденных записей time
time Информация о времени выполнения запроса Название
тип Описание ID
int Иидентификатор подразделения NAME
string Название подразделения SORT
int Поле сортировки подразделения PARENT
int Идентификатор родительского подразделения HTTP-статус: 20х, 40x, 50x Описанные ниже ошибки могут возникнуть при вызове любого метода Статус Код

## ТАБЛИЦЫ

Таблица 0:
sort
string | Название поля, по которому нужна сортировка. Сортировка работает по полям:
ID — идентификатор подразделения

SORT — порядок сортировки
PARENT — идентификатор родительского отдела
UF_HEAD — идентификатор пользователя-руководителя отдела
order
string | Направление сортировки:
ASC — по возрастанию
DESC — по убыванию

int | Фильтр по идентификатору подразделения
NAME
string | Фильтр по имени подразделения. Указывается полное название подразделения
SORT
int | Фильтр по полю сортировки
PARENT
int | Фильтр по идентификатору родительского подразделения
UF_HEAD
int | Фильтр по идентификатору руководителя подразделения
START
integer | Параметр используется для управления постраничной навигацией.
Размер страницы результатов всегда статичный: 50 записей.
Чтобы выбрать вторую страницу результатов, необходимо передавать значение 50. Чтобы выбрать третью страницу результатов — значение 100 и так далее.
Формула расчета значения параметра start:
start = (N-1) * 50, где N — номер нужной страницы.
