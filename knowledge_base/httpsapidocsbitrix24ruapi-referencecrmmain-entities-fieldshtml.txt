## ОПИСАНИЕ
Внимание! Более полный перечень полей приводится на страницах методов, возвращающих описание полей объектов. Такие методы имеют название crm.название_объекта.fields. Описание полей возвращает метод crm.deal.fields Название
тип Описание Чтение Запись ID

crm_status Идентификатор стадии. Возможные значения: Да Да STAGE_SEMANTIC_ID

char Флаг новой сделки (сделки в первой стадии) Да Нет IS_RECURRING

char Повторное обращение Да Да PROBABILITY
integer Вероятность Да Да CURRENCY_ID
crm_currency Идентификатор валюты сделки Да Да OPPORTUNITY

crm_contact Идентификатор привязанного контакта. Устаревший. Сохраняется для совместимости Да Да CONTACT_IDS
crm_contact Идентификатор привязанного контакта. Множественный. При использовании crm.deal.update и crm.deal.add можно подать массив контактов. В методах crm.deal.list и crm.deal.get поля нет и необходимо использовать crm.deal.contact.items.get для получения списка контактов. Для очистки поля используйте crm.deal.contact.items.delete, для замены значения используйте crm.deal.contact.items.set Да Да QUOTE_ID
crm_quote Идентификатор квоты. Только для чтения. Устаревший. Используйте метод crm.quote.list с фильтром по сделке Да Нет BEGINDATE

string Дополнительно об источнике. Текстовое поле Да Да ADDITIONAL_INFO
string Дополнительная информация Да Да LEAD_ID
crm_lead Идентификатор привязанного лида Да Нет LOCATION_ID

string Идентификатор источника данных. Используется только для привязки к внешнему источнику Да Да ORIGIN_ID
string Идентификатор элемента в источнике данных. Используется только для привязки к внешнему источнику Да Да UTM_SOURCE
string Рекламная система (Yandex-Direct, Google-Adwords и другие) Да Да UTM_MEDIUM
string Тип трафика: CPC (объявления), CPM (баннеры) Да Да UTM_CAMPAIGN
string Обозначение рекламной кампании Да Да UTM_CONTENT
string Содержание кампании. Например, для контекстных объявлений Да Да UTM_TERM
string Условие поиска кампании. Например, ключевые слова контекстной рекламы Да Да PARENT_ID_xxx
crm_entity Поля связей. Если на портале есть смарт-процессы, связанные с контактами, то для каждого такого смарт-процесса существует поле, хранящее связь между этим смарт-процессом и контактом. Само поле хранит идентификатор элемента такого смарт-процесса. Например, поле PARENT_ID_153 — связь со смарт-процессом entityTypeId=153, хранит идентификатор элемента этого смарт-процесса, связанного с текущим контактом Да Да LAST_ACTIVITY_BY
string Идентификатор пользователя, ответственного за последнюю активность в этом лиде (например, создавшего новое дело в лиде) Да Да LAST_ACTIVITY_TIME
datetime Время последней активности Да Да UF_CRM_ххх Пользовательские поля Да Да
