# Зарегистрировать сервис
URL: https://apidocs.bitrix24.ru/api-reference/ai/ai-engine-register.html

## ОПИСАНИЕ
Scope: ai_admin Кто может выполнять метод: администратор REST-метод для добавления пользовательского сервиса. Метод регистрирует engine, при повторном вызове обновляет его. Это не совсем место встройки, так как endpoint партнера должен следовать четким форматам. Параметр Описание С версии name
unknown Осмысленное и короткое название, которое будет появляться в интерфейсе пользователя. code
unknown Уникальный код engine category
unknown Может быть либо text (генерация текста), либо image (генерация картинок), либо audio (распознавание текста). completions_url
unknown endpoint для обработки пользовательского запроса. settings
unknown Тип AI (см. описание ниже). Необязательный. 23.800 Метод в случае успеха вернёт ID добавленного engine. Массив параметров: Параметр Описание С версии code_alias
unknown Тип AI. Доступны значения: ChatGPT (Open AI), GigaChat (Сбер), YandexGPT (Яндекс) (по умолчанию ChatGPT). model_context_type
unknown Тип подсчета контекста. Доступны значения: token - токены, symbol - символы. По умолчанию token. model_context_limit
unknown Объем контекста (по умолчанию 16К). Перед отправкой вам запроса пользователя, проверяется лимит контекста согласно типу подсчета. Внимание! В скрипте все в едином потоке кода, это для примера. В режиме production необходимо вынести строчки кода в отдельную часть. Шаблон для создания пользовательского endpoint можно использовать для кастомизации собственного сервиса. Ответ ожидается в течение определенного времени, потом колбек становится невалидным. Внимание! Помимо кода ответа, в случае успешной генерации, обработчик должен обязательно возвращать json_encode(['result' => 'OK']). Когда вы работаете с категорией провайдера audio, в ключе prompt вы получаете массив, который включает в себя следующие элементы: В ответе провайдеру уходят так же дополнительные поля: Поля Описание С версии auth Данные по авторизации, 23.600.0 payload_raw Сырое значение промпта (при использовании Copilot там будет символьный код использованного промпта) 23.600.0 payload_provider Символьный код провайдера препромпта (при использовании Copilot там будет prompt). 23.600.0 payload_prompt_text Если payload_provider = prompt, будет содержаться сырая инструкция препромпта. Это не обработанный препромпт для самостоятельного анализа. Подробнее в документации по промптам. 23.800.0 payload_markers Массив дополнительных маркеров от пользователя (original_message, user_message, language), использованный при формировании промпта. Подробнее в документации по промптам. 23.800.0 payload_role Роль (инструкция), использованная при формировании промпта. В GPT-подобных системах вы должны отправлять эту роль как системную в массиве сообщений. 23.800.0 context Массив предшествующих сообщений в хронологическом порядке. Например, список комментариев к посту. Первым в таком списке контекста считается авторское сообщение (сам пост). Важно: Объем контекста, отправляемого вашему провайдеру, зависит от указанного вами объема провайдера и типа подсчета (подробнее в документации по провайдеру). По умолчанию метод подсчета "токены", объем 16K. Отправлять контекст непосредственной нейронке вы должны только если передан параметр collect_context, равный true (1). В остальных случаях он передается как доп.информация на ваше усмотрение. 23.800.0 max_tokens Максимальное число лексем. Параметр контролирует длину вывода. Необязательный. temperature* Температура. Параметр контролирует случайность вывода (низкие значения делают вывод более сфокусированным и детерминированным). Обязательный. * - Обязательные параметры Пример Допустим, к вам приходит (помимо прочей информации) три массива данных. В этом случае, результирующий массив мы получаем как: Как использовать примеры в документации

## ПРИМЕРЫ КОДА

try
{
 const response = await $b24.callMethod(
  'ai.engine.register',
  {
   name: 'Ivanov GPT',
   code: 'ivanov_gpt',
   category: 'text',
   completions_url: 'https://antonds.ru/ai/aul/completions/',
   settings: {
    code_alias: 'ChatGPT',
    model_context_type: 'token',
    model_context_limit: 16*1024,
   },
  }
 );
 
 const result = response.getData().result;
 console.info(result);
}
catch( error )
{
 console.error(error);
}
---
