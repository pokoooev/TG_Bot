## ОПИСАНИЕ
Scope: pay_system Кто может выполнять метод: администратор CRM (право «Разрешить изменять настройки») Метод изменяет платежную систему. Обязательные параметры отмечены * Название
тип Описание ID*
sale_paysystem.ID Идентификатор платежной системы FIELDS
object Объект, содержащий новые значения полей (подробное описание приведено ниже) Название
тип Описание NAME
string Название платежной системы DESCRIPTION
string Описание платежной системы PERSON_TYPE_ID
sale_person_type.id Идентификатор типа плательщика BX_REST_HANDLER
sale_paysystem_handler.CODE Код REST-обработчика, указанный при добавлении обработчика методом sale.paysystem.handler.add ACTIVE
string Индикатор активности платежной системы. Возможные значения: LOGOTYPE
string Логотип платежной системы (картинка в формате Base64) NEW_WINDOW
string Флаг, отвечающий за настройку «Открывать в новом окне». Возможные значения: XML_ID
string Внешний идентификатор платежной системы Как использовать примеры в документации HTTP-статус: 200 Название
тип Описание result
boolean Результат обновления платежной системы time
time Информация о времени выполнения запроса HTTP-статус: 400, 403 Название
тип Описание error
string Строковый код ошибки. Может состоять из цифр, латинских букв и знака подчеркивания error_description
